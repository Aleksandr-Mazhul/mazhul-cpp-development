# Лабораторная работа 4: Рваная матрица (Jagged Matrix)

Проект демонстрирует работу с рваной (ступенчатой) матрицей в C++: динамическое выделение памяти, вывод, сортировка строк по критерию и освобождение ресурсов.

## Что делает программа

- Создает рваную матрицу с `rowCount` строками.
- Для каждой строки задает свою длину (`rowSize[i] = i + 1`).
- Заполняет элементы по формуле `matrix[i][j] = i + j`.
- Сортирует строки по убыванию количества положительных элементов.
- Печатает матрицу до и после сортировки.
- Запускает встроенные тесты.

## Ключевые функции

- `allocateJaggedMatrix(int**& matrix, int*& rowSize, int rowCount)`  
  Выделяет память под массив строк и массив длин строк, затем заполняет значения.
- `printJaggedMatrix(int** matrix, const int* rowSize, int rowCount)`  
  Выводит каждую строку с учетом ее индивидуальной длины.
- `criterionPositive(const int* row, int length)`  
  Возвращает число положительных элементов в строке.
- `sortRows(int** matrix, int* rowSize, int rowCount)`  
  Пузырьковой сортировкой переставляет строки так, чтобы сначала шли строки с большим количеством `> 0`.
- `freeMatrix(int** matrix, const int* rowSize, int rowCount)`  
  Освобождает всю выделенную память (`delete[]` для каждой строки, затем для `matrix` и `rowSize`).

## Тесты

В `tests.cpp` есть проверки основных сценариев:
- сортировка строк по количеству положительных;
- случай с одной строкой;
- строка нулевой длины;
- смешанные данные (положительные/отрицательные).

Точка входа для тестов: `runAllTests()`.

## Структура файлов

- `jagged_matrix.h` - объявления функций API.
- `jagged_matrix.cpp` - реализация выделения, сортировки, печати и освобождения.
- `main.cpp` - демонстрация работы и запуск тестов.
- `tests.cpp` - набор проверок корректности.

## Идея лабораторной

Главная цель - отработать ручное управление памятью и понять, как хранить таблицы, где строки имеют разную длину. Это полезный шаг перед изучением контейнеров STL (`std::vector<std::vector<int>>`) и более безопасных техник управления ресурсами.
