# cpp-class-relations-lab

Практическая работа по теме отношений между классами в C++.

## Цель

Закрепить на практике:
- композицию;
- ассоциацию (невладеющие связи);
- dependency (зависимость через параметры методов);
- выбор между `unique_ptr`, `shared_ptr`, `T*`, `T&`.

## Условие

Реализовать модель игры с классами:
- `Game`
- `Level`
- `Player`
- `Enemy`
- `Weapon`
- `ScoreManager`

Требования к связям:
- `Game` управляет набором `Level` (владение).
- `Level` управляет набором `Enemy` (владение).
- `Player` и `Enemy` используют `Weapon`, одно оружие может быть общим.
- `Player` работает со `ScoreManager`, но не хранит его как поле.
- `Player` знает текущую цель `Enemy`, `Enemy` знает `Player`.

## Что уже подготовлено

В папке есть шаблоны классов:
- `Game.h`
- `Level.h`
- `Player.h`
- `Enemy.h`
- `Weapon.h`
- `ScoreManager.h`
- `main.cpp` (демонстрационный сценарий)

Во всех методах стоят заглушки с `throw std::logic_error(...)`.

## Задача студента

1. Реализовать методы в заголовочных файлах.
2. Сохранить типы отношений и указателей по смыслу.
3. Добиться корректного запуска `main.cpp` без исключений.
4. Кратко пояснить, почему выбран именно такой тип связи в каждой паре классов.

## Рекомендуемый порядок реализации

1. `Weapon`
2. `ScoreManager`
3. `Player`
4. `Enemy`
5. `Level`
6. `Game`
7. Проверка через `main.cpp`

## Подсказка по типам

- `unique_ptr` — один владелец (композиция).
- `shared_ptr` — общее владение.
- `T*` — невладеющая ссылка, может быть `nullptr`.
- `T&` — невладеющая ссылка, не может быть `nullptr`.

## Ожидаемый результат

После реализации программа должна:
- создать игру, уровень, игрока и врагов;
- назначить цель игроку;
- начислить очки через `ScoreManager`;
- вывести финальные данные в консоль.
