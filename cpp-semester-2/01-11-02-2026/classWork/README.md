## 🛠 Подробный разбор кода (`ex11.cpp`)

### 1. Строгие перечисления (`enum class`)

```cpp
enum class TrafficLight : std::uint8_t {
    Red, Yellow, Green,
};
```

- Использование `enum class` предотвращает неявное приведение к `int` и коллизии имен (например, просто `Red`
  недоступен, нужно писать `TrafficLight::Red`).
- **Оптимизация памяти:** Явно указан базовый тип `std::uint8_t` (занимает всего 1 байт вместо стандартных 4 байт для
  `int`).
- **Функция конвертации:** `trafficLightToString` переводит значения светофора в строку через `switch`. Обратите
  внимание: тут нет блока `default`. Это хорошая практика при работе с `enum class`, так как современные компиляторы (
  GCC/Clang) выдадут предупреждение (`warning`), если вы добавите новый цвет, но забудете обработать его в `switch`.

### 2. Анализ последовательностей (`getTypeOfSequence`)

Функция собирает статистику переходов между соседними элементами массива:

- `less` — если был переход по возрастанию (`array[i] < array[i + 1]`)
- `more` — если был переход по убыванию (`array[i] > array[i + 1]`)
- `equals` — если соседние элементы равны.

**⚠️ Заметки и риски кода:**

1. **Выход за границы массива:** В цикле `for (std::size_t i = 0; i < length; ++i)` идет обращение к `array[i + 1]`. На
   последней итерации (когда `i == length - 1`), код попытается прочитать `array[length]`, что является выходом за
   пределы выделенной памяти (UB). *Исправление: нужно итерироваться до `length - 1`.*
2. **Отсутствие return на всех путях:** Сейчас функция возвращает пустую строку только в одном случае (строго монотонное
   возрастание), но поведение для других комбинаций (или если массив пуст) не определено. Функция требует доработки!
