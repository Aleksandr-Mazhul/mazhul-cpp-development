# Dynamic Array (Кастомный вектор)

Этот проект представляет собой реализацию структуры данных "Динамический массив" на языке C++ — упрощенного аналога стандартного `std::vector`.

## Описание

Класс `Array` управляет динамически выделяемой памятью. При создании он выделяет базовый массив определенной вместимости (`capacity`). При превышении этой вместимости класс автоматически перевыделяет память с учетом коэффициента роста (`growFactor`), копирует туда старые элементы и удаляет предыдущий массив. Это позволяет скрыто от пользователя управлять выделением памяти и работать с массивом как с бесконечным контейнером.

## Функциональность

- **Добавление элементов (`add(int element)`)**: 
  Добавляет новый элемент в конец массива. Если текущий массив заполнен (`length == capacity`), размер массива автоматически увеличивается (вместимость умножается на `growFactor`).
- **Геттеры**:
  - `getSource()` — возвращает указатель на сырой (raw) массив данных, что позволяет работать с элементами напрямую через указатель.
  - `getLength()` — возвращает текущее количество добавленных в массив элементов (не путать с `capacity` — общей вместимостью).
- **Вывод в поток**: 
  В проекте перегружен оператор `<<` для потока вывода `std::ostream`. Это позволяет легко выводить содержимое массива на экран одной строкой: `std::cout << myArray << std::endl;`, получая на выходе красиво отформатированный список вида ` Elements of array [1 2 3 ]`.

## Структура файлов

- `Array.h` — заголовочный файл с объявлением класса `Array`, его полей (таких как `capacity`, `length`, `growFactor`, `source`) и методов.
- `Array.cpp` — файл с реализацией методов класса `Array` и перегрузкой оператора вывода в поток.
- `main.cpp` — точка входа, где создается объект `Array` и демонстрируется процесс заполнения и вывода массива.

## Особенности реализации

- **Ручное управление памятью**: Использование `new` и `delete[]` для выделения и очистки памяти под массив `source`.
- **Инкапсуляция**: Внутренние параметры `capacity` и `growFactor` скрыты, пользователь оперирует только добавлением и чтением данных, не задумываясь об их перераспределении в памяти.
- **Дружественные функции**: Перегрузка оператора `<<` реализована как свободная функция (вероятно, объявленная через модификатор `friend` в классе `Array`), чтобы иметь возможность напрямую читать значения или использовать публичные геттеры.
