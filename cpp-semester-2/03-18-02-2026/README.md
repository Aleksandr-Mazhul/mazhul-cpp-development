## 🔍 Подробный разбор архитектуры (`FractionClass`)

Класс представляет собой дробь (`numerator/denominator`) и берет на себя всю низкоуровневую работу, избавляя
программиста от головной боли при расчётах:

### 1. Безопасность при инициализации (Конструктор)

* Запрет на нулевой знаменатель: если `denominator == 0`, то выбрасывается исключение `std::invalid_argument`.
* **Нормализация знака:** если передать минус в знаменатель (`1 / -2`), конструктор автоматически поднимет его вверх (
  `-1 / 2`).

### 2. Математические операции и сокращения

Каждая дробь всегда хранится в несократимом виде:

- При создании и любых сложениях вызывается приватный статический метод `reduce`. Инструмент `std::gcd` (наибольший
  общий делитель из библиотеки `<numeric>`) сам выполняет нужную математику.

### 3. Разные стили сложения (Гибкость интерфейса)

Класс демонстрирует сразу **три** различных способа реализации одной и той же операции сложения:

- ✍️ `addWith`: Мутирующий метод. Изменяет вызываемый объект (`fraction1.addWith(fraction2)`).
- ⚙️ `FractionClass::add(a, b)`: Чистая статическая функция. Возвращает новый объект.
- ➕ `operator+`: Перегруженный оператор. Внутри просто перенаправляет вызов на `add(...)`, что позволяет писать красивый
  и интуитивно понятный код: `FractionClass result = fraction1 + fraction2;`.

## ⚠️ Критические замечания к коду (Code Review)

1. **Неверное подключение файлов в `main.cpp`:**
   В `main.cpp` написано `#include "Rationalnumber.cpp"`. Это **плохая практика**! Директива `#include` должна
   использоваться только для `.h` файлов. Включая `.cpp`, мы рискуем получить ошибку `Multiple definition` (
   множественное определение), если проект будет собираться правильным CMake-ом, который компилирует каждый `.cpp` файл
   отдельно, а затем их линкует.
   _**Как исправить:**_ Измените инклуд на `#include "Rationalnumber.h"`.

2. **Неиспользуемое возвращаемое значение `addWith`:**
   Часто подобные "мутирующие" методы (или оператор `+=`) возвращают ссылку на себя (`*this`), чтобы можно было
   выстраивать методы в цепочки (Chaining): `f1.addWith(f2).addWith(f3);`. Сейчас этого сделать нельзя.
