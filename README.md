## 📘 Семестр 1: Фундаментальные основы и процедурное программирование

Первый семестр был полностью посвящен закладке железобетонного алгоритмического фундамента. Язык C++ рассматривался на низком уровне, приближенном к классическому Си, чтобы развить интуицию работы с памятью и понимание того, как данные представлены в компьютере.

### Ключевые освещенные темы:
1. **Базовая алгоритмизация и управляющие конструкции**: Работа с переменными, циклами (`for`, `while`, `do-while`), ветвлениями (`if/else`, `switch`). Манипуляция с числами и цифрами (построение перевертышей, счетчики).
2. **Анализ сложности и оптимизация**: Понимание временной сложности $O(N)$ и $O(N^2)$. Поиск способов сократить количество вложенных циклов.
3. **Статические массивы (1D)**: Алгоритмы циклического и логического сдвига элементов влево/вправо. Поиск экстремумов, уникальных элементов, вычисление средних и точек равновесия.
4. **Матрицы (2D Массивы)**: Выделение динамической и статической памяти. Обход по столбцам/строкам, перестановка секций, зеркалирование, замена диагоналей.
5. **Математические алгоритмы**: 
   - Написание функций разложения в Ряд Тейлора (`sin`, `cos`, `exp`) с заданным порогом погрешности `epsilon`.
   - Проверка чисел на специфические свойства: числа Армстронга, Харшада, ряды Фибоначчи, палиндромы, магические числа.
6. **Указатели (Pointers) и адресная арифметика**: Отказ от обращения по индексу `[i]` в пользу навигации по сырой памяти (указатели `begin`, `end`, операции `*ptr++`).
7. **Работа с текстом (`char*`)**: C-строки. Поиск слов, проверка ASCII-кодов, ручной парсинг без стандартных библиотек.

## 📙 Семестр 2: Продвинутое проектирование и C++11/14/17

Второй семестр смещает фокус с написания линейного кода на **архитектурное проектирование систем** и использование мощных возможностей современного стандарта C++. Здесь код становится гибким, переиспользуемым и безопасным.

### Ключевые освещенные темы:
1. **Объектно-Ориентированное Программирование (ООП)**:
   - Инкапсуляция полей, публичные/приватные интерфейсы (пример: проект `Counter`, `Timer`).
   - Конструкторы копирования, семантика перемещения (Move Semantics), перегрузка операторов присваивания и потокового вывода `operator<<`.
   - Исключения (Exceptions): валидация входных данных и выброс `std::invalid_argument`.
2. **Динамическое управление памятью C++ (Классы-обертки)**:
   - Реализация собственной версии `std::vector` (В проекте `Array.cpp`): динамическое реаллоцирование памяти с использованием `capacity` и `growFactor`. Избавление пользователя от необходимости ручного освобождения `delete[]`.
3. **Обобщенное программирование (Templates)**:
   - Создание типонезависимого кода. Папка `07-26-03-2026` демонстрирует универсальную шаблонную функцию `bubbleSort`, принимающую произвольные типы данных и функцию-ключ (Key Extractor).
   - Шаблонная кольцевая очередь `Queue<T, size>` (проект `queue`).
4. **Функциональные парадигмы в C++**:
   - Работа с `std::function`, передача лямбда-выражений (с захватом переменных `[capture]`) в качестве предикатов (`Predicate`), конвертеров (`Converter`) и компараторов (`Comparer`). 
   - Фильтрация массивов и предикативные архитектуры (`filterByPredicate`).
5. **Паттерны проектирования (Design Patterns)**:
   - Папка `09-10-04-2026/pizza/`: Классическая реализация порождающего паттерна **Builder (Строитель)** с плавным интерфейсом (Fluent Interface / Method Chaining) для конструирования сложных составных объектов (Пицца).
6. **Алгоритмы стандартной библиотеки (STL)**:
   - Моделирование Задачи Иосифа Флавия на `std::vector`.
   - Использование `std::sort` с кастомными лямбда-компараторами.
   - Подробное изучение макросов свойств типов, например `if constexpr (std::is_integral_v<T>)`.

## 🚀 Как запустить проекты

Для компиляции проектов рекомендуется использовать CMake. Убедитесь, что у вас установлен `cmake` и компилятор C++ (например, Xcode Command Line Tools для macOS, MSYS2 для Windows или GCC/Clang для Linux).

```bash
# 1. Клонируйте репозиторий
git clone <url-вашего-репозитория>
cd mazhul-cpp-development

# 2. Создайте директорию для сборки и перейдите в неё
mkdir build
cd build

# 3. Сконфигурируйте проект с помощью CMake
cmake ..

# 4. Соберите необходимый проект (таргет зависит от CMakeLists.txt)
cmake --build .

# 5. Запустите исполняемый файл
./name-of-executable
```

*Обратите внимание: большинство директорий (особенно во втором семестре) самодостаточны и могут быть скомпилированы напрямую вызовом `g++ main.cpp <остальные .cpp> -o main`.*

## 📈 Приобретенные Инженерные Навыки (Skills)

Выполнение десятков проектов и упражнений в этом репозитории позволило мне получить глубокие и профессиональные навыки:
*   **Управление ресурсами ОС**: Глубокое понимание разницы между Stack и Heap. Предотвращение утечек памяти (Memory Leaks) путем аккуратного балансирования операторов `new/delete`.
*   **Гибкая архитектура**: Понимание принципов DRY (Don't Repeat Yourself). Умение выносить общую логику в абстрактные шаблоны (`template`) и передавать конкретное поведение через предикаты/делегаты (`std::function`).
*   **Clean Code & Safety**: Избегание магических чисел (usoing `#define`, `const`, `constexpr`), использование модификатора `const` для защиты данных, правильное использование исключений для оповещения пользователя о невалидном состоянии программ.
*   **Тестирование**: Практика написания самописных Unit-тестов для каждого значимого модуля алгоритма для автоматической проверки краевых случаев.
